## Changes

### #160 — SymbolExporter.GetRomOffset()

- Replaced 5-way hardcoded platform name switch (nes/snes/gb/gba/default) with `PlatformResolver.Resolve()` + `analyzer.AddressToOffset()`
- Removed dead `ConvertSnesAddressToOffset()` method (~30 lines of LoROM/HiROM logic)

### #159 — Channel F PansyPlatformId

- Set `ChannelFProfile.PansyPlatformId` from `null` to `0x1f` (matching `Pansy.Core.PansyLoader.PLATFORM_CHANNEL_F`)
- All 11 platforms now have non-null PansyPlatformId values
- Updated test from `Assert.Null` to `Assert.Equal((byte)0x1f, ...)`

### #161 — DisassemblyEngine Platform Dispatch

- Added `IsValidAddress(uint)` and `GetTargetBank(uint, int)` to `IPlatformAnalyzer` interface
- Implemented in all 11 platform analyzers:
	- NES: `$8000`-`$ffff` valid, fixed bank for `$c000`+
	- SNES: always valid, bank from address bank byte
	- Game Boy: always valid, bank 0 for `$0000`-`$3fff`
	- GBA: always valid, bank from ROM address calculation
	- Atari 2600: `$f000`-`$ffff` valid
	- Atari Lynx: `$0200`-`$fbff` valid
	- Genesis/SMS/PCE/WonderSwan/ChannelF: always valid, return currentBank
- Replaced hardcoded string-based dispatch in `DisassemblyEngine.IsValidAddress()` and `GetTargetBank()` with analyzer method calls
- Updated 4 mock analyzer classes in test project
