## Root Causes Found

1. Signature-byte instructions (`cop`, `brk`, `wdm`) were vulnerable to assembler encoding drift.
2. Multi-bank long-address substitutions could collapse bank bytes during assembly.

## Fixes Implemented

1. Emit signature-sensitive opcodes as raw `.db` bytes in Poppy formatter.
2. Disable 24-bit label substitution in multi-bank mode (`address > 0xffff` is kept numeric).
3. Preserve explicit banked address text in formatter output path.
