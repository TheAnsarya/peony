## Key Changes

### IPlatformProfile Extensions

- Added `PoppyPlatformId` (required, no default) — maps platform to Poppy project string
- Added `DefaultRomExtension` (default impl) — derives from `RomExtensions[0]`
- All 11 profiles implement `PoppyPlatformId`

### Platform String Consolidation (#154)

- **ProjectWriter** — `NormalizePoppyPlatform()` and `GetRomExtension()` replaced with 2-line PlatformResolver lookups
- **ProjectScaffolder** — Same pattern applied
- **SymbolExporter** — 30+ entry `GetPansyPlatformId()` switch → single PlatformResolver lookup
- **HardwareIncludeGenerator** — String switch → PlatformResolver + PlatformId enum switch
- **PlatformMemoryMap** — All 3 public methods string switches → PlatformResolver + PlatformId enum switch
- 6 Registration classes gained Nexen-compatible aliases (superfamicom, gbc, gameboyadvance, mastersystem, pcengine, a26)

### Test Infrastructure

- `TestPlatformInitializer` — `[ModuleInitializer]` registers all 11 platforms in test assemblies
- `PlatformResolverCollection` — xUnit `[CollectionDefinition]` fixture prevents parallel test race conditions
- 6 test classes annotated with `[Collection("PlatformResolver")]`
