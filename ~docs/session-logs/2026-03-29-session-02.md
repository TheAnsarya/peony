### Behavior Changes

- Updated `src/Peony.Core/DisassemblyEngine.cs`:
	- Added explicit `SetStaticAnalysisEnabled(bool enabled)` opt-in switch.
	- Default flow now keeps static classification off (quarantined).
	- Pansy-derived entry hints are queued before CDL-derived hints.
- Updated `src/Peony.Cli/Program.cs`:
	- Added `--static-analysis` flag for explicit opt-in behavior.
	- Main `disasm` path now seeds entry points deterministically:
		- starts from the primary/first platform entry point,
		- then applies Pansy sub-entry/jump/xref targets,
		- then applies CDL sub-entry hints,
		- then adds remaining platform entry points.
