## Key Technical Discoveries

1. **Bank 0 ambiguity**: SNES LoROM bank 0 addresses (e.g., `$00:8000` = `0x008000`) are numerically ≤ `0xFFFF`, making them indistinguishable from 16-bit addresses. Fixed `GetTargetBank` to recognize `≥0x8000` as bank 0 code space.

2. **`AddressToOffset` bank detection**: Original code used `addrBank > 0` to detect 24-bit addresses, but bank 0 has `addrBank = 0`. Changed to `address > 0xffff` which correctly identifies 24-bit addresses regardless of bank number.

3. **Bank 7 mystery**: CDL shows 484 code bytes and 6 subs in bank 7, but Peony produces 0 blocks there. Needs further investigation — likely Pansy data type definitions overlapping with code regions. Could be a new issue.
