## Root Cause Fixed This Session

`PansyWriter.AddMultiTargetCrossReference` writes BOTH individual `CrossReference`
edges (step 4, uncapped) AND grouped `MultiTargetCrossReference` entries (step 4b,
capped). The cap was only on step 4b, so the legacy edges in step 4 bypassed it.
Fix: group step 4 by source address and apply the same per-source cap there.
