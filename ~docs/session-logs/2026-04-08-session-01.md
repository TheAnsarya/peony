# Session Log: 2026-04-08 Session 01

## Summary

Implemented the **Peony platform architecture separation** — a major structural refactor that introduces a plugin-based architecture for all 11 supported platforms, following the same pattern used in Poppy's `ITargetProfile` / `TargetResolver` system.

## Work Done

### Core Framework (Peony.Core/Platform/)

- **`PlatformId.cs`** — Enum identifying all 11 supported platforms
- **`IPlatformProfile.cs`** — Master plugin interface tying CPU decoder, analyzer, output generator, and asset extractors together
- **`IOutputGenerator.cs`** — Per-platform output generation interface
- **`IAssetExtractor.cs`** — Per-platform asset extraction interface
- **`PlatformResolver.cs`** — Central registry with name/alias/extension-based lookup (like Poppy's TargetResolver)

### SNES-Specific Output Generator

- **`SnesOutputGenerator.cs`** — SNES-specific `.pasm` output with:
	- `.system snes`, `.lorom`/`.hirom`/`.exlorom`/`.exhirom` directives
	- `.a8`/`.a16`/`.i8`/`.i16` state tracking via SEP/REP opcode detection
	- Per-bank formatting with correct `.org` addresses
	- Cross-reference comments, label substitution, typed comments

### Platform Profiles & Registrations (11 platforms)

Created `Profile.cs` + `Registration.cs` for each platform:

- NES, SNES, Game Boy, GBA, Atari 2600, Lynx, SMS, PCE, Genesis, WonderSwan, Channel F
- Each profile implements `IPlatformProfile` as a singleton
- Each registration maps aliases (e.g., "snes", "super nintendo", "super nes")
- Non-SNES platforms use `PoppyFormatter.Instance` as default output generator

### CLI Wiring (Program.cs)

- Added 11 `Registration.RegisterAll()` calls at startup
- Replaced 7 duplicated platform switch statements with `PlatformResolver.Resolve()` calls
- Fixed leftover `};` syntax artifacts from switch expression replacements

### PoppyFormatter Updates

- Added `IOutputGenerator` interface implementation
- Added `public static readonly PoppyFormatter Instance` singleton

### Tests (96 new tests)

- Created `PlatformResolverTests.cs` with comprehensive coverage:
	- Registration/lookup by name, alias, and extension for all 11 platforms
	- Pansy platform ID mapping verification
	- Singleton verification
	- Clear/unregistered platform error handling
	- SNES custom output generator vs generic formatter validation

### Documentation

- Updated README.md with new architecture section, 11 platforms, 1,489 test count
