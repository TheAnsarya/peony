## Summary

## Work Completed

### F8 CPU Decoder (`src/Peony.Cpu.F8/CpuF8Decoder.cs`)

- Full 256-opcode table implementation for the Fairchild F8 processor
- Custom `OpcodeKind` enum: Implied, Immediate, Absolute, Relative, RegLow4, RegLow3, RegLow4Imm, Undefined
- Complete opcode groups:
	- $00-$1F: LR register transfers, PK, shifts (SR/SL 1/4), LM, ST, COM, LNK, DI, EI, POP, INC
	- $20-$27: 2-byte immediate (LI, NI, OI, XI, AI, CI, IN, OUT)
	- $28-$2A: 3-byte absolute (PI, JMP, DCI) — big-endian addressing
	- $2B NOP, $2C XDC, $2D-$2F undefined
	- $30-$3F: DS r (scratchpad decrement/skip)
	- $40-$4F: LR A,r / $50-$5F: LR r,A (register-encoded)
	- $60-$67: LISU / $68-$6F: LISL (3-bit immediate)
	- $70 CLR, $71-$7F: LIS (4-bit immediate)
	- $80-$87: BT (branch on test), $88-$8E: memory ops (AM, AMD, NM, OM, XM, CM, ADC)
	- $8F: BR7, $90: BR, $91-$9F: BF (branch on false)
	- $A0-$AF: INS / $B0-$BF: OUTS (port operations)
	- $C0-$CF: AS / $D0-$DF: ASD / $E0-$EF: XS / $F0-$FF: NS (scratchpad ALU)
- `Decode()`, `IsControlFlow()`, `GetTargets()` fully implemented
- `FormatRegister()` maps r0-r9, hu(10), hl(11), s(12), i(13), d(14), r15
- Static helpers: `GetFullLrMnemonic()`, `GetFullShiftMnemonic()`

### Channel F Platform Analyzer (`src/Peony.Platform.ChannelF/ChannelFAnalyzer.cs`)

- `FrozenDictionary<uint, string>` I/O port register labels (PORT0-PORT5)
- `Analyze()` identifies ROM type by size (BIOS ROM 1, 2K/6K Cartridge, etc.)
- `GetMemoryRegion()` maps: BIOS/Cart ROM ($0000-$17FF), RAM ($2800-$2FFF), VRAM ($3000-$37FF), I/O ($3800-$38FF)
- `GetEntryPoints()` returns $0800 for carts, $0000 for BIOS-only
- `AddressToOffset()` handles full images and cart-only ROMs
- No bank switching (standard Channel F)
